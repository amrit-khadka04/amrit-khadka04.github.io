Transformer Insight

What are Input Embeddings?

In transformers, input embeddings convert discrete tokens (words, subwords) into continuous dense vectors. This is the first step in the transformer pipeline:

  • Tokenization — Text is split into subword tokens using BPE, WordPiece, or SentencePiece
  • Vocabulary Lookup — Each token maps to an index in a learned vocabulary (typically 30K–100K tokens)
  • Dense Vectors — The embedding layer projects each index into a d-dimensional space (e.g., 768 or 4096 dimensions)
embedding = nn.Embedding(vocab_size, d_model) x = embedding(token_ids) # [batch, seq_len, d_model]

Positional Encoding Spire

LLM Architecture Deep Dive — Order Matters

Sinusoidal Encoding

The original "Attention Is All You Need" paper used fixed sinusoidal functions: PE(pos, 2i) = sin(pos / 10000^(2i/d)). These create unique position signatures using different frequency waves, allowing the model to learn relative positions.

Learned Positional Embeddings

Models like BERT and GPT-2 use learned positional embeddings — trainable vectors for each position. Simple but limited to a fixed context length seen during training.

RoPE (Rotary Position Embedding)

Modern LLMs like LLaMA use Rotary Position Embeddings that encode position by rotating the query and key vectors. RoPE naturally captures relative positions and can extrapolate to longer sequences.

Why Position Matters

Unlike RNNs, transformers process all tokens in parallel — they have no inherent sense of order. Without positional encoding, "The cat sat on the mat" and "mat the on sat cat The" would be identical.

My Research Connection

At the UKP Lab, I work directly with these architectural choices. When fine-tuning Qwen-VL for visual reasoning, understanding how positional encodings interact with vision tokens versus text tokens is critical for multimodal alignment.

How Positional Encoding Works

Positional encoding adds position information to the embedding vectors, so the model knows where each token is in the sequence:

x = token_embedding + positional_encoding # Shape: [batch, seq_len, d_model]

The key insight: by using sinusoidal functions at different frequencies, the model can attend to relative positions. For position p and dimension i:

PE(p, 2i) = sin(p / 100002i/d)
PE(p, 2i+1) = cos(p / 100002i/d)

Multi-Head Self-Attention

The attention mechanism is the heart of the transformer. It computes how much each token should "attend to" every other token:

Attention(Q, K, V) = softmax(QKT / √dk) · V

Key concepts:

  • Q (Query) — What I'm looking for
  • K (Key) — What I have to offer
  • V (Value) — The actual information
  • Multi-Head — Run attention h times in parallel with different learned projections, then concatenate
Q = x @ W_q # [batch, seq, d_k] K = x @ W_k # [batch, seq, d_k] V = x @ W_v # [batch, seq, d_v] scores = (Q @ K.T) / sqrt(d_k) attn = softmax(scores) @ V

The Feed-Forward Network

After attention, each token passes through a position-wise Feed-Forward Network (FFN) — two linear transformations with a nonlinearity:

FFN(x) = W2 · GELU(W1 · x + b1) + b2
  • First layer projects from d_model to d_ff (typically 4× larger)
  • Activation — ReLU (original) or GELU/SwiGLU (modern LLMs)
  • Second layer projects back from d_ff to d_model
ffn = nn.Sequential( nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model) )

The FFN is where most of the model's "knowledge" is stored — factual associations, patterns, and learned transformations.

Layer Normalization & Residual Connections

Two critical mechanisms that make deep transformers trainable:

output = LayerNorm(x + Sublayer(x))
  • Residual Connections — Add the input directly to the output of each sub-layer (attention or FFN). This creates "skip connections" that allow gradients to flow freely during backpropagation
  • Layer Normalization — Normalizes activations across the feature dimension, stabilizing training by ensuring consistent scale
  • Pre-Norm vs Post-Norm — Modern LLMs (GPT-3, LLaMA) use Pre-Norm (normalize before sublayer), while the original transformer used Post-Norm
# Pre-Norm (modern) x = x + attention(layer_norm(x)) x = x + ffn(layer_norm(x))

Output Projection & Softmax

The final layer of the transformer maps hidden states back to the vocabulary space:

P(next_token) = softmax(Wout · h + b)
  • Linear Projection — Maps the d_model hidden state to vocab_size logits
  • Softmax — Converts logits into a probability distribution over the vocabulary
  • Weight Tying — Many models share weights between the input embedding and output projection (reduces parameters)
  • Sampling Strategies — Top-k, top-p (nucleus), temperature scaling control generation diversity
logits = hidden_state @ embedding.weight.T probs = F.softmax(logits / temperature, dim=-1) next_token = top_p_sample(probs, p=0.9)
